# Как вносить изменения

## Цель

Этот репозиторий ведётся внутри Gulian Digital LLC. Цель процесса - сохранить его легко запускаемым локально, безопасным для GitHub и удобным для внутреннего review.

## Локальный запуск

1. Используй Node.js `22`.
2. Скопируй `.env.example` в `.env`.
3. Установи зависимости через `npm install`.
4. Для локальной разработки используй `npm run dev:http` или `npm run dev:stdio`.
5. `.env` подхватывается автоматически из корня репозитория.
6. Для ручной проверки webhook route используй URL с `?token=$WEBHOOK_SHARED_SECRET`.

## Что проверить перед Pull Request

1. Выполни `npm run check`.
2. Выполни `npm run build`.
3. Убедись, что в коммите нет реальных секретов, токенов и клиентских данных.
4. Обнови `README.md` или `.env.example`, если изменились поведение или конфигурация.
5. Обнови `CHANGELOG.md`, если есть новое поведение, исправление, переработка, security-изменение или новый конфиг.

## Рекомендации по веткам и коммитам

- Одна ветка должна решать одну задачу.
- Пиши понятные commit message.
- Не смешивай рефакторинг и изменение поведения без необходимости.
- Внешние Pull Request не предполагаются; работа идёт через внутренние ветки и review owner.

## Что ожидается от Pull Request

- Коротко опиши, что изменилось и зачем.
- Отдельно отметь изменения в конфиге, auth, storage или webhook-логике.
- Если изменилось поведение системы, добавь короткую запись в `CHANGELOG.md`.
- Если изменение видно пользователю, добавь заметку о ручной проверке.
- Если правка касается `.env`, Docker или CI, явно укажи, как повторить запуск.

## Безопасность

- Не коммить реальный `.env`.
- Не коммить amoCRM токены и секреты.
- С webhook-примерами работай только в очищенном и безопасном виде.
